Sequences

If you use EE enough you will find yourself doing the same thing over and over again. Trigger sequences are a way to automate your EE workflow by automatically running a list of Trigger commands. These can be used for something as simple as a pre-launch sequence of commands, to a list of commands that builds an entire blog.

Sequence Structure

Sequences are stored as plain text files with a .txt extension, and are places in the sequences folder within Trigger. Using text files allows you to easily transport frequently used sequences from site to site. Each file has a simple naming structure:

seq.pre_launch.txt

In the above example, the text in between the prefix (seq.) and suffix (.txt) is the name of the sequence. Each sequence needs a unique name.

Sequences have a standard internal structure that contains both data about the sequence and the Trigger commands:

title: Pre-Launch
desc: Things to do before Launch.
author: Adam Fairholm
author url: http://www.addictaddons.com

TRIGGER SEQUENCE START
ee : templates : sync
ee : site : clear cache
ee : site : disable op
ee : site : online
TRIGGER SEQUENCE END

As you can probably guess, the top part of the sequence contains the data. Here are the data points you can provide:

Data Piece Description
title The title of the sequence.
desc A short description of what the sequence does.
author Your name.
author url Optional. A full web address for you.
created Optional. A timestamp for when this sequence was created.

Running Sequences

There are two ways to run a sequence. One is from the Trigger Sequences panel where you can see a list of sequences and run them. Trigger will show you the lines you are about to run. The next is with the sequence driver, where you can run it like this:

ee : seq : run (pre_launch)

Sequence Pre-Checks

Once a sequence starts, it just rams through all of its commands, so it is a good idea to check to make sure it is okay to run the commands before you do. You can do this by adding check lines before the sequence. These look like this:

#check snippet header does not exist

This will look for a snipper called "header" and will not allow you to run the sequence if it finds one. Right now you can look to see if snippets ("snippet"), global variables ("global_var"), and templates ("template") exist.
