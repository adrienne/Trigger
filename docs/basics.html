Trigger Basics

Drivers

Drivers are a key concept to understand before we start with the structure of Trigger commands. Drivers are basically grouped functions that you can access via Trigger. There is a site driver (for accessing basic site and config commands), a template driver (for manipulating templates), and many more. You can even create your own drivers to do what you want. More on that later, but let's first get into how to actually use Trigger...

Trigger Commands

Trigger works like this: you give it a text command, and it does something for you, then gives you a response. When you first open up Trigger, you'll see the command window with a simple prompt:

ee :

This is the base of a Trigger command. It means that EE is ready to do what you want it to. If you make a command and there is no error and no ee prompt, then something has gone wrong!

Two Part Commands

A Trigger command comes in two types: a two part command and a three part command. A two part command is just the ee root and something else. For example:

ee : server_path

This is an example of a variable call (we'll get into those later), but it is an example of something you can call in the 2nd command slot.

In general, things you do with two-slot commands are pretty basic. Here's what you can do with two slots:

  • Run Trigger system commands
  • Return the value of Trigger system variables
  • Load a Driver

Most of the things you'll do in Trigger utilize three part commands.

Three Part Commands

Three part commands allow you to access functionality in drivers. They look something like this:

ee : site : disable op

The above command calls the site driver, and calls the command disable op. This disables the output profiler.

Many Trigger driver functions need data to process, and you can pass that data in between parenthesis. For example:

ee : groups : new (about)

The above will create a new group (with an index.html template) called about. If more than one variable is needed, you can separate them via commas:

ee : globals : new (twitter_handle, addictaddons)

The above creates a new global variable with the value of "addictaddons".

That's it! Working Trigger is actually pretty simple. Once you get the hang of it, you can do things a lot quicker and even automate a lot of what you do with EE.

Getting Out of a Driver

If you are working in a driver and want to go back to the ee : context to use another driver, just type in "root" and it will take you where you need to go!
