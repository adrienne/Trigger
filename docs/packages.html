Packages

Sequences are great, but what if you want to add a template or a snippet with some data to it? For instance, if you really wanted to automate the creation of a blog, then you'd need template files that can be inserted.

Enter packages. Packages are convenient ways to hold templates, snippets, and sequences all in one folder for easy portability.

Packages Structure

Packages sit in the packages folder within trigger and have a set structure. Here is an example package called "blog":

blog
- package.txt
- seq.build_blog.txt
- snippets
	- sidebar.txt
	- post.txt
- templates
	- blog_index.html
	- archive.html
	- single.html

You can have as many sequences as you'd like in your package folder.

Package.txt

Package.txt contains some basic info about your package. Here is an example:

name: Blog Structure
description: Creates a basic blog
author: Addict Add-ons
author url: http://www.addictaddons.com
copyright: 2011 Addict Add-ons

For your package, you cab provide the following data:

Data Piece Description
name The name of the package.
desc A short description of the package.
author The package author's name.
author url Optional. A full web address for the package author.
copyright Optional. A copyright notice for the package.

Loading Package Resources

When you have a Trigger command like template : new, you can pass data to the template to populate it. You can load in a package resource (from anywhere, no just in package sequences) using the following syntaxt:

p.package_name/templates/template_name

The above will grab the content from the template folder of the specified package (it will automatically append .html for template, .txt for everything else) and use that data. Note that you don't have to just use the snippets and templates folder, you can specify any folder you'd like.

Viewing Package Contents

To get a quick overview of the contents for a package, just go to the packages pane of Trigger. You can click on a package name to view package details and a file list.
